Git 学习总结

发表于2022-11-07|更新于2022-11-07
|阅读量:

1.Git 设置用户签名

签名的作用是区分不同的操作者身份

git 必须设置用户签名，否则无法提交代码

这里设置的用户签名和 github 等账号没有任何关系

本地的 git 设置文件在 用户\用户名\.gitconfig

2.Git的基本操作流程

2.1初始化本地库

在文件夹中想要使用 git 管理就需要使用 git 初始化本地库

执行命令后会在文件夹中创建一个 .git 的隐藏文件夹，一般情况下不要修改该文件夹中的文件

2.2查看本地库状态

查看当前本地库中文件状态

当创建了新的文件后就会显示文件的状态

2.3添加暂存区

把文件添加到暂存区是提交的必要步骤

新建文件或修改文件都需要再次将文件添加到暂存区

提交本地库

3.1查看版本信息

3.2版本穿梭

通过修改指针的指向进行版本的更换。

4.分支操作

4.1基础操作

4.2合并分支

需要注意的是合并分支是将命令中的分支合并到当前所在的分支

分支合并只会修改当前所在分支的文件，被合并的分支的文件没有改动

4.3冲突合并

合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。git无法替我们决定使用哪一个。必须认为解决新代码内容。

合并冲突的文件需要手动修改，修改完以后再进行添加追踪和提交到本地仓库

解决冲突后提交的本地仓库的命令中不要加文件名

注意需要把git个的提示字符也一并手动删除掉，不然会留存到文件中
